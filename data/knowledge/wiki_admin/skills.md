# 위키 업데이트 에이전트 스킬

에이전트가 지식베이스를 업데이트하거나 건강 상태를 점검할 때 사용하는 절차 문서.

---

## 스킬 1: 새 지식 추가 (ingest)

새 출처(PDF, URL, 스크랩 데이터)에서 지식을 추출해 페이지를 생성하는 절차.

**트리거**: "~에 대해 배웠다", "새 자료가 추가됐다", 출처 파일이 `data/source/`에 추가됨

**절차**:
1. `schema.md` 읽기 — 네이밍·frontmatter 규칙 확인
2. `index.md` 읽기 — 기존 관련 페이지 존재 여부 확인
3. 기존 페이지 있음 → 해당 페이지 업데이트 + `last_updated` 갱신
4. 기존 페이지 없음 → 새 파일 생성, frontmatter 작성, `index.md`에 항목 추가
5. 연관 페이지에 `[[위키링크]]` 추가

**금지**: 출처 없는 내용 추가, importance 임의 상향

---

## 스킬 2: 위키 건강 점검 (lint)

고아 페이지, 저중요도 문서, 깨진 위키링크를 탐지하고 정리 제안.

**트리거**: "위키 점검해줘", "고아 페이지 찾아줘", 정기 실행

**점검 항목**:

| 항목 | 판정 기준 | 권장 조치 |
|------|-----------|-----------|
| 고아 페이지 | `index.md`에 없거나 어떤 페이지에서도 `[[링크]]`되지 않음 | 삭제 또는 연결 추가 |
| 저중요도 과잉 | `importance: low` 페이지가 도메인 내 30% 초과 | 병합 또는 삭제 검토 |
| 깨진 위키링크 | `[[페이지명]]`이 실제 파일과 불일치 | 링크 수정 또는 파일 생성 |
| 오래된 페이지 | `last_updated`가 6개월 이상 경과 + 법령·수치 포함 | 재검토 표시 추가 |
| 중복 페이지 | 제목·내용이 80% 이상 유사한 페이지 2개 이상 | 병합 |

**출력**: `wiki_admin/lint_report.md` 업데이트

---

## 스킬 3: 페이지 병합 (merge)

중복·유사 페이지를 하나로 통합하는 절차.

**절차**:
1. 두 페이지 내용 비교, 고유 정보 식별
2. 메인 페이지(importance 높은 쪽)에 고유 내용 통합
3. 서브 페이지 삭제 — `wiki_admin/log.md`에 기록
4. 삭제된 페이지를 참조하던 `[[링크]]` 모두 메인 페이지로 교체
5. `index.md` 업데이트

---

## 스킬 4: 도메인 간 이동 (reclassify)

페이지가 잘못된 도메인 폴더에 있을 때 재분류.

**절차**:
1. 올바른 도메인 폴더로 파일 이동
2. frontmatter `domain` 필드 수정
3. `index.md`에서 항목 위치 수정
4. 해당 페이지를 참조하는 `[[링크]]`는 파일명만 참조하므로 수정 불필요 (Obsidian 자동 해결)
